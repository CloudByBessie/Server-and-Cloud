## 🌩️ Cloud Tech — Origin Story

<img width="300" alt="CloudTech logo with electric design" src="https://github.com/user-attachments/assets/19e41a0e-c356-4a23-9fdc-74fe36012644" />


🛡️ **Role: Defensive Security / Blue Team Operations**

Cloud Tech wasn’t started in a boardroom. It started with a problem.

In the early days of cloud adoption, companies rushed to migrate everything—applications, identities, entire infrastructures—without fully understanding the risks. Misconfigured servers, exposed directories, weak identity controls… attackers didn’t need zero-days.

They just needed patience.

That’s where Cloud Tech comes in.

Founded by a security engineer focused on modern infrastructure defense, Cloud Tech began as a personal initiative to break, understand, and secure modern cloud environments—starting with the backbone of enterprise infrastructure:

👉 **Windows Server & Active Directory**

Instead of trusting systems, Cloud Tech tests them.  
Instead of assuming security, Cloud Tech proves it.

Their mission is simple:

🛡️ **Build secure environments by thinking like an attacker**

## ⚡ Thunderbyte — Origin Story

<img width="300" alt="ThunderByte logo with dynamic lightning bolt" src="https://github.com/user-attachments/assets/2e9f514c-f0b9-4a9f-913c-418b944c6ad2" />


⚡ **Role: Offensive Security / Red Team Operations**

Thunderbyte is a name that has existed since the earliest days of computing.

Long before cloud infrastructure…  
Before modern security frameworks…  
Thunderbyte built its reputation on one thing:

👉 **Staying ahead of everyone else**

They didn’t just compete — they *bit into* the competition.

Through decades of evolution—from on-prem networks to early internet systems—Thunderbyte adapted faster than most organizations could defend.

Their philosophy:

> “If it can be built, it can be broken.”

But when cloud technology emerged, something changed.

For the first time, Thunderbyte encountered an environment that wasn’t predictable.

- Identity became the new perimeter  
- Infrastructure became abstract  
- Traditional attack methods didn’t always work  

And for the first time…

⚡ **Thunderbyte had a real challenger**

## ⚔️ The Scenario

This environment mirrors a small enterprise network under active attack conditions.

### 🌩️ Cloud Tech (Blue Team)

- Builds and secures infrastructure  
- Manages Active Directory & identity  
- Detects and responds to threats  

### ⚡ Thunderbyte (Red Team)

- Enumerates and exploits weaknesses  
- Targets misconfigurations  
- Attempts privilege escalation and persistence  

---

## 🎯 Lab Objective

This lab is built and tested in a virtualized environment using multiple machines (Domain Controller, Client, and Attacker).

Technologies involved include Active Directory, DNS, Windows Server, and Kali Linux.

This project simulates real-world enterprise security challenges in a controlled cyber battlefield.

You will:

- 🏗️ Build an enterprise-like environment  
- 🔍 Identify vulnerabilities  
- ⚔️ Simulate attacks  
- 🛡️ Implement defenses
